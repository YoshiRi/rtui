# Context-Sensitive Search 仕様書

## 概要

`Ctrl+F` はフォーカスのあるパネルに対してのみ検索を起動する。  
パネルをまたいで検索状態が干渉することはない。

```
左パネルにフォーカス中 → Ctrl+F → 左パネル内の検索バーをアクティブにする
Infoパネルにフォーカス中 → Ctrl+F → Infoパネル内の検索バーをアクティブにする
下パネルにフォーカス中 → Ctrl+F → 下パネル内の検索バーをアクティブにする
```

---

## 共通ルール

| 項目 | 仕様 |
|------|------|
| マッチ方式 | 大文字小文字を区別しない部分一致 |
| 検索テキスト | Enter適用後も検索バーに残す（フィルタ中であることを明示） |
| フィルタリセット | エンティティ切り替え時にそのパネルのフィルタをリセット |
| 検索バーの配置 | 各パネルの最上部（1行固定） |

---

## パネル別仕様

### 1. 左パネル（RosEntityListPanel）

**既存動作を維持する。変更なし。**

| キー | 動作 |
|------|------|
| `Ctrl+F` または `/` | 左パネル上部の検索バーにフォーカス |
| 文字入力 | **リアルタイム**でエンティティ一覧を絞り込む |
| `Enter` | ツリーにフォーカスを戻す（フィルタは維持） |
| `Esc` | 検索テキストをクリア → 全件表示 → ツリーにフォーカス |

- フィルタ対象: エンティティのフルパス名
- エンティティを選択しても左パネルのフィルタは**維持される**（意図的に絞り込んだ状態を保つ）
- 他パネルとの干渉: なし

---

### 2. Infoパネル（RosEntityInteractivePanel）

| キー | 動作 |
|------|------|
| `Ctrl+F` | Infoパネル上部の検索バーにフォーカス |
| 文字入力 | 変化なし（タイプ中はプレビューしない） |
| `Enter` | フィルタ適用 → カーソルを最初のマッチに移動 → 検索バーのテキスト維持 |
| `Esc` | 検索テキストをクリア → 全リンク表示 → Infoパネルにフォーカスを戻す |

**フィルタ動作:**
- フィルタ対象: 各リンクの `label`（トピック名・ノード名・型名を含む文字列）
- マッチしたリンクのみ表示（非マッチは非表示、セクションヘッダも対象リンクがゼロなら非表示）
- マッチがゼロ件の場合: `(no matches)` を表示
- `↑↓` カーソル移動はフィルタ後の表示リンクのみを対象とする

**リセットタイミング:**
- 別のエンティティを選択したとき（左パネルで別ノード/トピックを選択）

---

### 3. 下パネル — NodeParamPanel（パラメータ一覧）

| キー | 動作 |
|------|------|
| `Ctrl+F` | パラメータパネル上部の検索バーにフォーカス |
| 文字入力 | 変化なし |
| `Enter` | フィルタ適用 → DataTable を絞り込み表示 |
| `Esc` | テキストクリア → 全行表示 → DataTable にフォーカスを戻す |

**フィルタ動作:**
- フィルタ対象: Parameter列（パラメータ名）と Value列（値文字列）の両方
- マッチしない行は非表示

**リセットタイミング:**
- 別のノードを選択したとき

---

### 4. 下パネル — NodeHzPanel（Hz一覧）

| キー | 動作 |
|------|------|
| `Ctrl+F` | Hz一覧パネル上部の検索バーにフォーカス |
| 文字入力 | 変化なし |
| `Enter` | フィルタ適用 → DataTable を絞り込み表示 |
| `Esc` | テキストクリア → 全行表示 → DataTable にフォーカスを戻す |

**フィルタ動作:**
- フィルタ対象: Topic列（トピック名）
- `f` キーによる Dir フィルタ（Sub/Pub/All）と**独立して組み合わせ可能**
  - 例: Dir=Sub かつ Topic に "camera" を含む行のみ表示

**リセットタイミング:**
- 別のノードを選択したとき

---

### 5. 下パネル — RosTypeDefinitionPanel（型定義テキスト）

| キー | 動作 |
|------|------|
| `Ctrl+F` | 型定義パネル上部の検索バーにフォーカス |
| 文字入力 | 変化なし |
| `Enter` | ハイライト適用（テキスト自体の行は非表示にしない） |
| `Esc` | テキストクリア → ハイライト解除 → 型定義パネルにフォーカス |

**ハイライト動作:**
- マッチ箇所を `[on yellow]...[/on yellow]` でハイライト
- テキスト全体は常に表示（行の非表示はしない）
- ステータスバーにマッチ件数を表示: 例 `(3 matches)`

**リセットタイミング:**
- 別のエンティティを選択したとき

---

### 6. 下パネル — TopicMonitorPanel（Hz + Echo）

**検索非対応。**

- Hz 表示は1行のみ → 検索の意味がない
- Echo（RichLog）は流れるログであり、フィルタ・ハイライトともに不適

---

## 実装方針

### Ctrl+F バインドの移動

現在 `screens.py` の Screen レベルに `Binding("ctrl+f", "focus_search")` があり、常に左パネルを対象にしている。  
これを**削除**し、各パネルウィジェット自身に `Binding("ctrl+f", "focus_search")` を追加する。

Textual のバインド優先順位: **フォーカスの近いウィジェットが優先**されるため、  
Infoパネルにフォーカスがあれば Infoパネルの `Ctrl+F` が、左パネルにフォーカスがあれば左パネルの `Ctrl+F` が発火する。

### 各パネルへの変更

| ウィジェット | 変更内容 |
|---|---|
| `RosEntityListPanel` | `Binding("ctrl+f", ...)` を Screen から移管するのみ。ロジック変更なし |
| `RosEntityInteractivePanel` | 上部に `Input` 追加、`_apply_filter()` / `_clear_filter()` 実装 |
| `NodeParamPanel` | 上部に `Input` 追加、`refresh_params()` にフィルタ適用ロジック追加 |
| `NodeHzPanel` | 上部に `Input` 追加、`_visible_rows()` に検索フィルタ条件追加 |
| `RosTypeDefinitionPanel` | 上部に `Input` 追加、`update_content()` にハイライトロジック追加 |
| `TopicMonitorPanel` | 変更なし |
| `screens.py` | Screen レベルの `Ctrl+F` バインドを削除 |

### 検索バーのUI

各パネルで統一したレイアウト:

```
┌────────────────────────────────────────┐
│ Search: [________________________] ×   │  ← 1行、常時表示
├────────────────────────────────────────┤
│ （パネル本体）                          │
└────────────────────────────────────────┘
```

- 検索バーは常時表示（hidden にしない）
- `Ctrl+F` でフォーカスが移るだけ
- 空文字列の場合はフィルタなし（全件表示）と同義
